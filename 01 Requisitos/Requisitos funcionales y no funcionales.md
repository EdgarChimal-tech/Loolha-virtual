# Requisitos Funcionales

<table>
<thead>
<tr>
<th>RF</th>
<th>Requisito</th>
<th>Prioridad</th>
<th>Justificación</th>
</tr>
</thead>
<tbody>
<tr>
<td>1</td>
<td>Inicio de sesión del administrador</td>
<td>Alta</td>
<td>Es fundamental para que el administrador pueda acceder al sistema y gestionar el menú.</td>
</tr>
<tr>
<td>2</td>
<td>Registro de nuevos platillos o bebidas</td>
<td>Alta</td>
<td>Es una función central del sistema de administración de menú.</td>
</tr>
<tr>
<td>3</td>
<td>Consulta de productos registrados</td>
<td>Media</td>
<td>Es importante para gestión, pero no crítico para la operación inmediata; se puede visualizar, pero no es imprescindible.</td>
</tr>
<tr>
<td>4</td>
<td>Actualización de productos existentes</td>
<td>Alta</td>
<td>Necesario para mantener el menú actualizado y correcto.</td>
</tr>
<tr>
<td>5</td>
<td>Eliminación de productos del menú</td>
<td>Alta</td>
<td>Es esencial para retirar productos que ya no están disponibles.</td>
</tr>
<tr>
<td>6</td>
<td>Cierre de sesión del administrador</td>
<td>Media</td>
<td>Importante por seguridad, pero el sistema puede funcionar, aunque no se cierre sesión correctamente.</td>
</tr>
<tr>
<td>7</td>
<td>Consulta del menú por parte del usuario</td>
<td>Alta</td>
<td>Es la función principal del sistema para los usuarios; si no funciona, el sistema pierde su propósito.</td>
</tr>
<tr>
<td>8</td>
<td>Consulta de información (“Acerca de”)</td>
<td>Baja</td>
<td>Información adicional que no afecta la operación principal del sistema.</td>
</tr>
<tr>
<td>9</td>
<td>Envío de mensajes de contacto</td>
<td>Baja</td>
<td>Función útil para comunicación, pero no indispensable para la operación del menú.</td>
</tr>
</tbody>
</table>

# Requisitos No Funcionales

<table>
<thead>
<tr>
<th>RNF</th>
<th>Requisito</th>
<th>Justificación</th>
</tr>
</thead>
<tbody>
<tr>
<td>1</td>
<td>Usabilidad de la interfaz</td>
<td>Los elementos de navegación deberán estar claramente identificados y su comprensión no deberá requerir conocimientos técnicos ni avanzados.</td>
</tr>
<tr>
<td>2</td>
<td>Rendimiento de carga</td>
<td>El sistema deberá cargar la información del menú en un tiempo máximo de 3 segundos.</td>
</tr>
<tr>
<td>3</td>
<td>Seguridad visual en el inicio de sesión</td>
<td>Durante el proceso de inicio de sesión, el campo de contraseña deberá ocultar los caracteres ingresados mostrando únicamente el símbolo “*”.</td>
</tr>
<tr>
<td>4</td>
<td>Cambios del contenido</td>
<td>El sistema deberá permitir agregar, actualizar o eliminar productos del menú mediante la interfaz de administración sin necesidad de modificar directamente el código fuente.</td>
</tr>
<tr>
<td>5</td>
<td>Compatibilidad con navegadores</td>
<td>El sitio web deberá ser completamente funcional y visualmente estable en Google Chrome, Microsoft Edge, Mozilla Firefox, Opera y Brave.</td>
</tr>
<tr>
<td>6</td>
<td>Diseño responsivo</td>
<td>El sitio web deberá adaptarse correctamente a diferentes resoluciones y tamaños de pantalla, incluyendo computadoras, tablets y dispositivos móviles.</td>
</tr>
</tbody>
</table>
